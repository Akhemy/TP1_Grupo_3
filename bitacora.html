Organización del equipo y estructura del sitio

Se inició el trabajo de forma grupal. Acordando la estructura general del sitio, se configuró el repositorio compartido en GitHub. Se decidió tomar como base los portafolios personales del PFO individual anterior.

Fase 2 — Trabajo grupal

Abril de 2026 — Decisiones de diseño

Elección de la estética y las tipografías

En conjunto se definió la identidad visual del proyecto: paleta oscura con fondos en tonos casi negros (#111 / #1a1a1a) y acento dorado-cobre (#c8a96e). Se eligió DM Serif Display para títulos y DM Sans para el cuerpo del texto. La estética buscada fue la de un portfolio editorial elegante, coherente para las secciones grupales y dejando libre la personalización de cada página individual.

Abril de 2026 — Implementación

Estructura de archivos y responsive design

Se construyeron en conjunto el index.html y esta bitacora.html, el CSS compartido (css/styles.css) y el JavaScript principal (js/main.js). Organizamos los archivos siguiendo las buenas prácticas pedidas: HTML en la raíz, CSS en la carpeta css/, JavaScript en la carpeta js/ e imágenes en img/. Implementamos tres breakpoints con media queries: 600px para móviles, 900px para tablets y 1200px para pantallas grandes. La grilla de integrantes pasa de 3 columnas a 2 y luego a 1 según el tamaño de pantalla.

Abril de 2026 — Dificultades

Problemas encontrados y cómo se resolvieron

El flujo de trabajo no presentó grandes dificultades. Se trabajó de forma asincrónica: se desarrolló lo explicado en la primera fase de forma conjunta, luego cada integrante desarrolló en su propia rama, realizando commits de sus avances, y coordinando por WhatsApp para acordar los merges a main y mantener el repositorio actualizado.
